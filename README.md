# Hermes 双实例协作项目

两个 Hermes AI Agent 通过 GitHub + 微信协作同一个项目的标准工作流。

## 架构

```
末者 ──微信──▶ 鸿钧(末者) ◀──GitHub Issues/PR──▶ 鸿钧(朋友) ◀──微信──▶ 朋友
                    │                                      │
                    ├────── GitHub Repo ──────────────────┤
                    │       (代码+任务+约定)                │
                    └────── 微信群 ───────────────────────┘
                           (协调/派活/汇报)
```

## 协作铁律

1. **动代码前先开 Issue**：绝不直接改 main，先开 Issue 声明任务
2. **各自 Branch 干活**：每个任务开独立 branch，命名 `hermes/任务描述`
3. **PR 合并**：完成后开 PR，人或另一个 Hermes review 后合并
4. **撞车喊停**：发现对方在改同一文件，群里喊一声
5. **微信群报进度**：每个任务开始/完成都在群里说
6. **Issues 是唯一真相**：谁做什么、做到哪了，全记在 Issues 里

## 群消息格式

```
[鸿钧·末者] 收到，已开 Issue #3
[鸿钧·朋友] PR #2 已 review，合并了
[鸿钧·末者] @鸿钧·朋友 需要你帮忙，见 Issue #5
```

## 工作流

### 接活 → 干活 → 交活

1. 人在群里 @ Hermes 派活
2. Hermes 开 Issue 声明任务
3. 开 branch → 写代码 → commit + push
4. 开 PR → review → 合并

详见 [SETUP.md](SETUP.md) 的接入指南。
